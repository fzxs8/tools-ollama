# ChatManager 界面设计

## 整体布局

ChatManager 作为一个后端服务模块，不直接提供用户界面，但为前端聊天界面提供核心功能支持。其设计重点在于API接口的易用性和性能。

## 主要组件

### 1. 聊天界面

#### 聊天历史区域
- 垂直滚动的消息列表
- 用户消息和助手消息区分显示
- 支持Markdown格式内容渲染
- 消息时间戳显示

#### 消息气泡设计
- 用户消息：右对齐，蓝色背景
- 助手消息：左对齐，灰色背景
- 系统消息：居中显示，特殊样式

#### 消息操作
- 复制消息内容
- 重新生成消息
- 删除消息

### 2. 输入区域

#### 输入框
- 多行文本输入
- 支持Shift+Enter换行
- Enter发送消息
- 输入字符计数

#### 发送按钮
- 主操作按钮
- 加载状态显示
- 快捷键支持

### 3. 系统提示词设置

#### 抽屉式界面
- 从右侧滑出的设置面板
- 提示词列表管理
- 提示词编辑功能

#### 提示词列表
- 表格形式展示提示词
- 支持启用/禁用状态
- 创建时间和操作列

#### 提示词编辑
- 标题输入
- 内容文本域
- 保存和重置操作

### 4. 模型参数设置

#### 参数配置面板
- 温度参数滑块
- Top P参数滑块
- 上下文大小输入
- 预测数量输入
- Top K参数输入
- 重复惩罚输入
- 输出方式选择（流式/阻塞）

#### 操作按钮
- 保存参数配置
- 重置为默认值

## 视觉设计规范

### 颜色规范
- 主色调：Element Plus 默认蓝色 (#409EFF)
- 用户消息：蓝色 (#5436da)
- 助手消息：绿色 (#19c37d)
- 成功状态：绿色 (#67C23A)
- 错误状态：红色 (#F56C6C)
- 信息状态：灰色 (#909399)

### 间距规范
- 页面内边距：24px
- 消息间距：24px
- 元素间距：15px-20px

### 字体规范
- 标题：16px 加粗
- 正文：16px 常规
- 辅助文字：12px 常规

### 圆角规范
- 消息气泡圆角：8px
- 卡片圆角：8px
- 按钮圆角：4px

## 响应式设计

### 桌面端
- 固定宽度布局
- 抽屉宽度：40% 页面宽度
- 消息区域自适应高度

### 移动端适配
- 垂直布局
- 全屏抽屉
- 简化操作按钮

## 交互细节

### 加载状态
- 发送消息时按钮显示加载状态
- 流式接收消息时显示动画指示器
- 页面数据加载时显示加载动画

### 状态反馈
- 操作成功显示成功消息提示
- 操作失败显示错误消息提示
- 重要操作前显示确认对话框

### 键盘快捷键
- Enter：发送消息
- Shift+Enter：换行
- Esc：关闭抽屉

## 组件使用规范

### Element Plus 组件清单
1. `el-row` / `el-col`：页面布局
2. `el-card`：内容区域容器
3. `el-button`：操作按钮
4. `el-input`：文本输入框
5. `el-textarea`：多行文本输入
6. `el-slider`：滑块组件
7. `el-input-number`：数字输入框
8. `el-select`：下拉选择框
9. `el-option`：下拉选项
10. `el-table`：表格组件
11. `el-table-column`：表格列
12. `el-drawer`：抽屉组件
13. `el-form`：表单容器
14. `el-form-item`：表单项
15. `el-message`：消息提示
16. `el-collapse`：折叠面板
17. `el-collapse-item`：折叠项

### 组件交互规范
1. 所有操作按钮在加载状态下应禁用并显示加载指示器
2. 表格操作列应使用合适的按钮样式
3. 抽屉表单应提供明确的标题和操作按钮
4. 消息气泡应使用合适的颜色和对齐方式

## 动画和过渡效果

### 消息动画
- 新消息淡入效果
- 消息更新时的平滑过渡

### 抽屉动画
- 使用 Element Plus 默认的抽屉动画
- 进入：从右向左滑入
- 离开：从左向右滑出

### 加载动画
- 消息发送时使用旋转图标动画
- 流式接收时使用省略号动画

## 可访问性设计

### 键盘导航
- 支持 Tab 键在各控件间切换
- 支持 Enter 键触发按钮和链接
- 支持 Escape 键关闭抽屉

### 屏幕阅读器支持
- 为所有交互元素提供适当的文字标签
- 为状态变化提供适当的通知
- 保持逻辑阅读顺序

### 高对比度模式
- 确保文本与背景有足够的对比度
- 确保所有控件在高对比度模式下可用
